## 测试结果与问题发现

### 问题1：附件上传导致400错误

**现象**：上传 `测试附件/` 中的PDF文件，后端返回 `400 Bad Request`。

**根因分析**：  
测试附件目录下的所有PDF文件均带有 `%TSD-Header`（可信存储设备加密头），PyMuPDF无法在Python进程中直接解析这类加密文件。文本提取返回空字符串，导致 `doc_texts` 为空，进而命中"请提供至少一种输入"的校验逻辑抛出400。

**解决方案**：修改 `backend/routers/intake.py` 的文件处理逻辑：
- 原来：文本提取失败 → 直接报错
- 现在：文本提取失败 → 保留文件名，向 `doc_texts` 注入占位提示文字 `[文件内容：xxx（文本提取失败，文件可能为扫描件或加密格式）]`，让Agent知晓文件已提交但无法解析
- 效果：用户不再收到400错误，Agent仍能处理其他输入维度

**备注**：TSD加密PDF在浏览器端通过TSD驱动可正常解密，是客户端功能，Python后端无法绕过。

---

### 问题2：更新操作中 `old` 值始终为 null

**现象**：解析结果中更新操作的 `changed_fields[字段].old` 始终为 null，前端无法显示"从X改为Y"的变更对比。

**根因分析**：
1. AI模型（qwen3.5-plus）读取 `existing_targets` 提示词填充 `old` 值不稳定
2. 字段名映射不一致：DB中部分字段以 `revenue_yuan` 存储，AI收到的existing_targets展示格式不统一
3. 原始代码 `_DISPLAY_FIELDS` 列表使用中文label，AI无法直接映射到字段key

**解决方案（双保险机制）**：

**前端提示词优化**（`prompts/intake_agent_prompt.py`）：
```
- **更新操作的 old 值**：已有标的库每条记录包含了主要字段的当前值（格式为 `字段名=当前值`），
  字段名与可提取字段清单中的字段名完全对应，请直接从中读取填入 old。
```

**后端Python保底补填**（`routers/intake.py`）：
```python
_FIELD_ALIASES = {
    "revenue_yuan": ["revenue_yuan", "revenue"],
    "net_profit_yuan": ["net_profit_yuan", "net_profit"],
    "valuation_yuan": ["valuation_yuan"],
}
for op in result.get("operations", []):
    if op.get("type") == "update" and op.get("bd_code"):
        existing = target_by_bd.get(op["bd_code"], {})
        for field_key, change in op.get("changed_fields", {}).items():
            if isinstance(change, dict) and change.get("old") is None:
                for candidate in _FIELD_ALIASES.get(field_key, [field_key]):
                    old_val = existing.get(candidate)
                    if old_val is not None and str(old_val) not in ("", "None", "nan"):
                        change["old"] = old_val
                        break
```

**效果**：即使AI未填充old，Python层也能从existing_targets中补填，确保变更对比完整。

### 问题3：标的库查询始终返回空

**现象**：即使已有多个项目，`_get_all_existing_targets()` 返回空列表，导致所有操作被判定为"新建"。

**根因分析**：系统使用SQLite DB（`reports.db`）存储项目记录，但当前DB为空，所有项目元数据仅存在于 `outputs/*.json` 文件中。

**解决方案**：在 `_get_all_existing_targets()` 中增加JSON文件fallback：
```python
if not targets:
    for meta_file in OUTPUT_DIR.glob("*.json"):
        if meta_file.stem.endswith("_chunks"):
            continue
        meta = json.loads(meta_file.read_text(encoding="utf-8"))
        if meta.get("bd_code") and meta.get("company_name"):
            targets.append(meta)
```

**效果**：现有13个项目的元数据均可被正确读取，更新场景可正常匹配。
