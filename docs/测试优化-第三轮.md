## 测试场景设计

边界条件测试旨在验证系统在非标准输入下的行为是否符合预期。

| 场景 | 输入类型 | 预期行为 |
|------|---------|---------|
| 空输入 | 无内容 | 400错误（"请提供至少一种输入"） |
| 非公司文本 | 纯文字（无企业信息） | 返回空operations，无异常 |
| 模糊简称匹配 | "盛航"（非全称） | 匹配到BD15059，判定为更新 |
| 同公司重复提及 | 同一公司出现3次 | 合并为1个operation |
| 多样金额格式 | "十五个亿"、"两千八百万" | 正确转换为数字 |

### ✅ 场景5：多样金额格式

**测试输入**：
```
某公司A估值十五个亿，营收两千八百万，净利润3.5亿。
```

**实际响应**：
```json

  "fields": {
```

**结论**：中文数字（"十五个亿"→1500000000，"两千八百万"→28000000）和小数（"3.5亿"→350000000）均正确转换为纯数字。AI的数字转换能力稳定。

## 额外发现：Chunker分块数量硬编码问题

**发现场景**：为测试索引生成，尝试对一个只有2个chunk的报告生成索引。

**问题**：`prompts/chunker_prompt.py` 中存在硬编码：
- "给定5个报告分块"（应为动态数量）
- "共5个元素"（应与实际chunks数量一致）

**影响**：当报告分块不足5个时，Chunker Agent可能生成包含空白元素的错误JSON数组，或因指令不一致而困惑。

**修复**（已应用到 `prompts/chunker_prompt.py`）：
```python
# 修改前
给定5个报告分块（chunks），为每个chunk生成15-22个搜索索引标签。
...
1. 输出严格JSON数组，共5个元素，与输入chunks顺序一一对应

# 修改后  
给定若干报告分块（chunks），为每个chunk生成15-22个搜索索引标签。
...
1. 输出严格JSON数组，元素数量与输入chunks数量完全一致，与输入chunks顺序一一对应
```

### 已解决的关键问题

1. **TSD加密PDF导致400** → 降级为占位文字，不中断流程
2. **更新操作old值始终null** → 双保险：提示词优化 + Python保底补填
3. **existing_targets为空（DB空）** → JSON文件fallback扫描
4. **description过长膨胀prompt** → 截断至100字
5. **Chunker硬编码5个分块** → 改为动态数量

### 尚存限制（非代码问题）

- **批量5家处理耗时90-120秒**：模型固有延迟，建议前端加超时提示
- **TSD加密PDF无法从Python直接解析**：需TSD客户端环境，浏览器端正常
- **Backend需手动重启**：所有代码变更需重启uvicorn才能生效
