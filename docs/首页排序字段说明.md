# 首页排序字段说明表（ReportsPage）

本文定义首页（标的管理列表）中“可点击排序”的字段、排序规则和后端实际映射，避免前后端不一致导致“点了排序但结果异常”。

## 总体规则

- 排序由后端 `GET /api/report/list` 执行，前端只传 `sort_by` / `sort_dir`。
- `sort_dir`：
  - `asc`：升序
  - `desc`：降序
- 空值统一排序到最后（无论升序/降序）。
- 时间字段按数据库原始值排序（包含时分秒），前端显示可只展示到“日”。

## 字段说明表

| 首页列名 | 前端列 key | 是否可排序 | 后端 sort_by | 排序规则 | 备注 |
|---|---|---:|---|---|---|
| 标的编码 | `bd_code` | 是 | `bd_code` | 文本排序，空值最后 | 常用主键型字段 |
| 标的项目 | `project_name` | 否 | - | - | 已禁用排序 |
| 标的主体 | `company_name` | 否 | - | - | 已禁用排序 |
| 行业 | `industry` | 否 | - | - | 多标签展示，不启用排序 |
| 评级 | `feasibility_rating` | 是 | `feasibility_rating` | 自定义顺序：A < B < C < D < E，空值最后 | 支持升序/降序 |
| 评级时间 | `feasibility_rating_at` | 否 | - | - | 当前不排序 |
| 报告生成状态 | `status` | 否 | - | - | 状态语义复杂，当前不排序 |
| 推介情况 | `referral_status` | 否 | - | - | 当前不排序 |
| 营业收入 | `revenue` | 否 | - | - | 当前不排序 |
| 报价/估值 | `offer_or_valuation` | 是 | `offer_or_valuation` | 数值排序：`COALESCE(offer_yuan, valuation_yuan)`，空值最后 | 按数学升降序，`0`按数学规则参与排序 |
| 报价/估值时间 | `offer_or_valuation_date` | 否 | - | - | 当前不排序 |
| 附件 | `attachments` | 否 | - | - | 当前不排序 |
| 推送状态 | `push_status` | 否 | - | - | 当前不排序 |
| 更新时间 | `updated_at` | 是 | `updated_at` | 时间排序（按秒），空值最后 | 前端显示到日，不影响排序精度 |
| 预估成本（默认隐藏） | `estimated_cost` | 是 | `estimated_cost` | 数值排序，空值最后 | 隐藏列，开启后可排序 |
| 净利润（默认隐藏） | `net_profit` | 否 | - | - | 按当前产品要求不提供排序 |

## 后端支持的 sort_by（当前）

后端白名单（`backend/routers/report.py`）当前支持：

- `created_at`
- `updated_at`
- `company_name`
- `project_name`
- `bd_code`
- `score`
- `rating`
- `status`
- `province`
- `industry`
- `estimated_cost`
- `feasibility_rating`
- `offer_or_valuation`

> 说明：即使后端仍保留部分字段支持（如 `company_name` / `project_name`），前端可以选择不暴露排序入口。

## 变更约定（后续新增排序字段时）

新增任一排序字段需同时完成以下三项：

1. 前端列定义中显式声明 `sortable: true`（或明确不排序）。
2. 前端 `sort_by` 与后端白名单保持一致（必要时做 key 映射）。
3. 在本文补充字段规则（特别是空值策略、数值/枚举顺序策略）。
