## 背景与问题

当前系统的录入和更新体验存在以下痛点：

1. **录入摩擦大**：用户需手动将非格式化内容（聊天记录、邮件、截图）逐字段填入表单
2. **更新代价高**：哪怕只改一个估值字段，也要重跑完整 6 步流水线（约 5~15 分钟）
3. **输入类型受限**：不支持图片、链接智能下钻
4. **多标的割裂**：一次沟通涉及多个标的时，需分别手动录入

**目标**：将录入入口改为 Agent 驱动，用户直接投入原始材料，AI 完成理解、判断和执行。

## 一、输入层

### 1.1 支持的输入类型

| 类型 | 处理方式 |
|------|---------|
| 非格式化文字 | 直接送 LLM 理解（粘贴聊天记录/邮件/备忘录） |
| 图片（聊天截图等） | 多模态 LLM 直接理解图片内容和上下文 |
| 文档（PDF/PPT/DOCX） | 现有解析链不变（PyMuPDF→pdfplumber���OCR） |
| 链接（官网/公告） | 网页 Agent 爬取 + 智能下钻 |

### 1.2 网页 Agent 下钻规则

- 爬取用户提交的链接页面后，AI 评估当前页面的信息完整度
- 若信息不足（如首页只有宣传语，缺少业务/团队/财务信息），自主决定下钻哪些子页面
- 判断依据写在提示词中，示例规则：
  - 优先下钻：「关于我们」「产品/服务」「团队/管理层」「投资者关系」
  - 不下钻：「联系我们」「隐私政策」「404页」
- **内容页最大爬取次数：默认 3 次**（可在设置中调整）
- 使用现有 `jina_reader` / `local_scraper` 工具链

## 二、录入 Agent

### 2.1 模型配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 模型 | `qwen3.5-plus` | 具备视觉识别能力，可直接理解截图 |
| API Base URL | 与全局配置相同 | 同一套 DashScope 接口 |
| API Key | 继承全局配置 | 可在设置页单独覆盖 |

### 2.2 Agent 职责

接收所有混合输入材料后，按以下步骤处理：

```
输入材料（文字+图片+文档+链接）
    ↓
① 理解所有材料内容
    ↓
② 识别涉及哪些标的（按公司名/BD编码匹配已有标的库）
    ↓
③ 对每个标的判断操作类型：
   - 新建：材料中出现的新公司，库中无匹配
   - 更新：材���中涉及已有标的的新信息
    ↓
④ 提取字段变化清单：
   - 新建：提取所有可识别字段
   - 更新：列出变化字段（旧值→新值）
    ↓
⑤ 输出结构化意图结果（JSON）
```

### 2.3 意图结果数据结构（示意）

```json
{
  "operations": [
    {
      "type": "create",
      "company_name": "张江生物科技有限公司",
      "fields": {
        "industry": "生物医药",
        "province": "上海",
        "is_listed": "未上市",
        "valuation_yuan": "200000000"
      },
      "source": ["聊天截图.png"]
    },
    {
      "type": "update",
      "bd_code": "BD00023",
      "company_name": "星辰网络科技",
      "changed_fields": {
        "valuation_yuan": { "old": "50000000", "new": "80000000" },
        "revenue_yuan":   { "old": "120000000", "new": "150000000" }
      },
      "source": ["财务报告.pdf"]
    }
  ]
}
```

## 三、执行模式

### 3.1 两种模式对比

| | 自动模式（默认） | 手动确认模式 |
|--|----------------|------------|
| 解析完成后 | 直接执行，完成后写更新记录 | 展示预览卡，用户确认后执行 |
| 适用场景 | 日常高频录入，信任 AI 判断 | 数据敏感，需人工核实 |
| 切换位置 | 录入页右上角勾选 + 设置页配置默认值 |

### 3.2 手动模式预览卡（自动模式跳过）

解析完成后在录入页面下方展开（不跳页）：

```

├─────────────────────────────────────────────────────┤
│ ① [新建]  张江生物科技有限公司                       │
│   行业: 生物医药 · 上海 · 未上市 · 估值: 2亿         │
│   [展开所有字段 ∨]   [编辑]   [移除]                │
│                                                     │
│ ② [更新]  BD00023 · 星辰网络科技                    │
```

## 四、执行流程

### 4.1 新建标的

走现有完整 6 步流水线，无变化：

```
Step1 提取 → Step2 联网研究 → Step3 生成报告 → Step4 字段回填 → Step5 分块索引 → Step6 FastGPT推送
```

### 4.2 轻量更新（不触发联网重调研）

```
字段变化确认
    ↓
Step3' 全文重写报告（变化字段 + 原报告全文 → LLM → 新完整报告）
    ↓
Step4  字段回填（同步元数据）
    ↓
Step5  重新分块与索引
    ↓
Step6  FastGPT 重推
```

**Step3' Prompt 策略：**

```
以下字段已更新：
- 估值：5000万 → 8000万
- 营收：1.2亿 → 1.5亿

原报告全文如下：
[原报告 Markdown 内容]

请基于上述变化，更新所有受影响的段落（包括但不限于：估值分析、
评分结论、核心摘要），其余段落保持原文不变，输出完整报告。
```

### 4.3 完整重新调研的触发

满足以下任一条件时，执行前提示用户选择：

| 触发条件 | 说明 |
|---------|------|
| 用户主动勾选 | 录入时可勾选「需要重新调研」 |
| 核心字段变化 | 变化字段包含：主营业务、股权结构（可在设置中配置清单） |
| 调研数据过期 | 上次联网调研距今超过 N 天（默认 90 天） |

**提示样式：**

```
⚠️ 本次更新涉及主营业务变化，建议重新调研以获得更准确结论。

[仅更新字段和报告]          [完整重新调研（约 10 分钟）]
```

### 4.4 调研数据复用提示

轻量更新执行时，进度区域显示：

```
⟳ 沿用上次调研数据（23 天前）
```

若超过有效期阈值，高亮显示：

```
⚠️ 沿用上次调研数据（91 天前，已超过有效期）— 建议完整重新调研
```

## 五、更新记录（新 Tab）

### 5.1 位置与定位

报告详情页 Tab 栏新增「更新记录」，与「版本历史」并列：

| Tab | 内容 | 可否回滚 |
|-----|------|---------|
| 版本历史（已有） | 报告全文快照 | ✅ 支持 |
| 更新记录（新增） | 每次操作的数据处理日志 | ❌ 只读 |

### 5.2 数据库表设计（新建）

表名：`intake_logs`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK | 自增 |
| `report_id` | TEXT FK | 关联 reports 表 |
| `log_type` | TEXT | `light_update` / `full_regenerate` / `create` |
| `trigger_reason` | TEXT | 触发原因描述 |
| `input_sources` | TEXT (JSON) | 输入材料列表（文件名/链接） |
| `changed_fields` | TEXT (JSON) | 字段变化清单（旧值→新值） |
| `steps_executed` | TEXT (JSON) | 实际执行的步骤列表 |
| `steps_skipped` | TEXT (JSON) | 跳过的步骤及原因 |
| `research_data_age_days` | INTEGER | 沿用的调研数据距今天数（轻量更新时） |
| `operator` | TEXT | 操作人用户名 |
| `created_at` | TEXT | 操作时间 ISO8601 |

## 六、设置页新增配置项

在现有「AI 设置」面板中新增「录入 Agent」配置区：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 录入 Agent 模型 | `qwen3.5-plus` | 需支持视觉（多模态） |
| API Key | 继承全局 | 可单独覆盖 |
| 网页下钻最大次数 | `3` | 每个链接最多追踪的子页面数 |
| 默认执行模式 | `自动` | 自动 / 手动确认 |
| 触发完整调研的核心字段 | `主营业务, 股权结构` | 变化时弹出提示 |
| 调研数据有效期（天） | `90` | 超过后高亮提示 |
