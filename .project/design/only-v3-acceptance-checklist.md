# Only V3 手工验收清单

> 适用版本：2026-04-17 之后的 Only V3 主线
> 目标：确认 only-v3 主线下，智能录入、任务状态、报告消费层和历史报告转 v3 更新链路均正常

---

## 1. 录入入口

- 打开首页，只能看到 `智能录入`，不能看到 Excel 上传、手动输入、模板下载
- 从报告列表进入新建或智能更新，均跳到 v3 录入页
- `ReportsPage` 顶部按钮文案为 `+ 智能录入`
- 报告列表操作菜单中原“生成报告”已替换为“智能更新”

## 2. 解析阶段

### 2.1 纯文本解析

- 输入一段明确包含公司/项目名称的文本，点击“开始解析”
- 解析按钮进入 `解析中 (xx%)`
- 页面持续显示解析阶段消息，不应长时间无反馈
- 解析结束后进入“解析结果确认”，而不是直接执行

### 2.2 文档/附件解析

- 上传 PDF / DOCX / PPTX / TXT / MD 任一文件，点击解析
- 解析状态应显示文件提取进度
- 解析完成后，确认卡中能看到关联附件名称
- 材料摘要不应为空；若识别为空，可手工补充后继续执行

### 2.3 多标的解析

- 输入包含 2 个以上标的的文本或附件
- 解析完成后应展示多张确认卡
- 每张卡均可单独勾选/跳过
- 执行时只应提交当前勾选的卡片

## 3. 确认层

- 新建项应允许编辑标的名称
- 更新项应展示匹配标的、匹配说明、置信度标签
- 中低置信匹配应显示提醒信息
- 编辑材料摘要后执行，报告内容应反映新的摘要输入
- 点击“重新解析”应清空本轮确认结果并重新走 parse

## 4. 执行与任务状态

- 点击“确认并执行”后，生成任务卡
- 任务卡中应持续显示步骤与 `message`
- SSE 断开后，任务不应直接误判为完成；应继续通过轮询更新
- 任务完成后，卡片应提供“查看报告”
- 点击“终止”后应要求二次确认

## 5. 报告消费层

### 5.1 新建报告

- v3 新建完成后可打开报告详情页
- 报告正文可正常查看
- 附件 Tab 可看到本次关联附件
- chunks 页可正常加载并编辑

### 5.2 更新报告

- 对已有报告执行一次智能更新
- 更新完成后仍进入原有 `report_id` 详情页
- 报告正文、chunks、附件、更新时间应同步刷新
- `intake_logs` 中应记录本次更新日志

### 5.3 下载与附属能力

- Markdown 下载可用
- PDF 下载可用
- FastGPT 推送入口可用
- 评级确认相关入口可用

## 6. 历史报告兼容

- 任选一个旧报告打开详情页
- 正文应可正常展示
- Markdown / PDF 下载不应报错
- 附件查看不应报错
- 若旧报告没有 `report_chunks`，chunks 页允许为空，但不应出现旧分块重建入口
- 若对旧报告发起一次智能更新，更新完成后应可正常加载 v3 chunks

## 7. 异常用例

- 空输入点击解析，应提示至少提供一种输入
- 解析超时或失败，应提示错误，不应卡死在 loading
- 更新项若缺少可执行目标，应在确认层明确标识，不应误执行
- 任务执行失败时，任务卡应显示失败 message

## 8. 通过标准

满足以下条件即可认为 Only V3 收敛完成：

- 用户侧已无 legacy 录入入口
- 解析、确认、执行三段链路均可稳定工作
- v3 任务 message 与进度可持续展示
- 新报告可正常消费；旧报告在完成一次智能更新后可正常消费 v3 chunks
- 未发现仍指向 `/api/upload`、`/report/generate`、`/report/batch-generate` 的前端入口
